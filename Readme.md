# Банковское приложение «Банковские гарантии»

**Цель:** Создание автоматизированной системы для оформления банковских гарантий

**AS-IS:** Заявка на получении банковской гарантии подается клиентом лично в отделении банка. Рассмотрение и согласование заявки производится специалистами вручную в CRM-системе банка. Клиент не может отслеживать статус заявки. Одобрение или отказ в услуге сообщается клиенту сотрудником банка по телефону или электронной почте.

**TO-BE:** Заявка на получение услуги «банковская гарантия» создается клиентом самостоятельно в приложении. Документы и данные о клиенте загружаются в систему и проверяются автоматически. Заявка проходит единый процесс согласования. Клиент в реальном времени может видеть статус своей заявки. Уведомление о выдаче или об отказе клиент получает в приложении.

## Бизнес-требования
1.	**БТ-1: Онлайн подача заявки**
Клиент имеет возможность удаленного оформления банковской гарантии, что увеличит число клиентов банка, а также сократит нагрузку на офис. 

2.	**БТ-2: Снижение загрузки сотрудников банка на 50%**
Время сотрудников может быть рационально распределено в соответствии с задачами банка.

3.	**БТ-3: Отслеживание клиентом статуса своей заявки**
Клиент может без обращения в офис узнавать о статусе рассмотрения своей заявки, что увеличивает лояльность клиентов и уменьшает нагрузку на сотрудников офиса.

4.	**БТ-4: Размещение ссылки на приложение на основных площадках электронных торгов по 44-ФЗ и 223-ФЗ**
Повышение уровня вовлеченности потенциальных клиентов банка

5.	**БТ-5: Электронный документооборот**
Система позволяет обмениваться с клиентом документами в электронном виде, что экономит время сотрудников банка и клиента

6.	**БТ-6: Автоматическая оповещение клиента о результатах**
Система оповещает клиента о результате проверки, т.о. 100% уведомлений доставляются корректно 



## Системные требования
### Функциональные требования
**Регистрация и аутентификация** 
1. ФТ-1: Система должна обеспечивать регистрацию пользователей по ИНН/ОГРН
2. ФТ-2: Система должна поддерживать аутентификацию пользователя по логину и паролю
3. ФТ-3: Система должна поддерживать аутентификация пользователя по ЭЦП
**Профиль клиента**
1. ФТ-4: Система должна позволять пользователю сохранять и редактировать данные об организации
2. ФТ-5: Система должна сохранять историю изменений профиля

**Заявка на банковскую гарантию**
1.	ФТ-6: Клиент должен иметь возможность оформить и подать заявку на получение банковской гарантии
2.	ФТ-7: Клиент должен иметь возможность приложить документы в формате JPEG, PDF (размер файла не более N Gb)
3.	ФТ-8: Клиент должен иметь возможность указать сумму и срок гарантии
4.	ФТ-9: Система должна иметь возможность работать с различными видами банковских гарантий
5.	ФТ-10: Система должна иметь возможность проверять введенные данные

**Согласование заявки** 
1.	 ФТ-11: Система должна поддерживать согласование заявки в несколько этапов
2.	 ФТ-12: Система должна фиксировать комментарии и решения.

**Управление рисками**
1.	ФТ-13: Система должна рассчитывать комиссию
2.	ФТ-14: Система должна проверять финансовые показатели клиента

**Выдача гарантии**
1.	ФТ-15: Система должна сформировать решение о выплате или об отказе в услуге
2.	ФТ-16: Система должна сформировать ответ на запрос клиента
3.	ФТ-17: Система должна направить ответ на запрос клиента в соответствии с шаблоном

**Интеграции**
1.	ФТ-18: Система должна быть синтегрирована с площадками электронных торгов (<https://www.rts-tender.ru/>,< https://www.roseltorg.ru/>, <https://etp.zakazrf.ru/>, <https://gz.lot-online.ru/etp_front/>, <https://www.astgoz.ru/page/index> )
2.	ФТ-19: Система должна быть синтегрирована с ФНС РФ
3.	ФТ-20: Система должна быть синтегрирована с криптопровайдером для использования ЭЦП (КриптоПро CSP)
4.	ФТ-21: Система должна быть синтегрирована с внешними сервисами ЭДО для использования ЭЦП (Контур.Диадок, СБИС, Такском)

### Нефункциональные требования:
**Безопасность**
1.	НФТ-1: Система должна блокировать попытки ввода пароля по IP-пользователя при неверном вводе пароля в количестве  >= 3 раз
2.	НФТ-2: Параметры безопасности соответствуют стандартам ЦБ РФ
3.	НФТ-3: Осуществляется защита от brute-force атак
4.	НФТ-4: Производится аудит и логирование всех операций
5.	НФТ-5: Система должна обеспечить шифрование данных
**Производительность**
1.	НФТ-6: Время отклика системы не должно превышать 3 сек
2.	НФТ-7: Система поддерживает одновременную работу N пользователей
 
**Надежность**
1.	НФТ-8: SLA системы должно составлять 99%
2.	НФТ-9: Выполнена репликация БД
3.	НФТ-10: Отсутствуют потери данных при сбое системы

**Удобство использования**
1.	НФТ-11: Интерфейс приложения должен быть адаптирован для мобильных устройств

**Ограничения системы**
1.	НФТ-12: Система работает только на территории РФ
2.	НФТ-13: Система работает в соответствии с 152-ФЗ «О персональных данных»
3.	НФТ-14: Срок хранения данных не менее 5 лет
